
## M1 — Planning Process

### What I did

Before writing any code, I planned the full app on paper first. I sketched out the screens by hand — the home dashboard with a 7-day chart, the add task form, the reminders page, and the history/analysis page. I also mapped out the user flow showing how a user opens the app, sees their tasks on the dashboard, adds new ones, gets reminded when something is coming up, and reviews their history on the analysis page.

From the sketches I identified the core features the app needs:

- Add tasks with a name, due date, and priority
- See all tasks on the dashboard and task table
- Delete and edit tasks
- Save tasks as JSON and CSV
- Dashboard with stats and graphs
- Reminders when a task is coming up or overdue
- Analysis page showing past activity

I then moved to digital wireframes, designing each page with ALU branding — navy blue, red, and ash — using real campus photography as hero images.
